================================================================================
  AUTHENTICATION AND AUTHORIZATION MODEL
================================================================================

  Auth Flow:
  1. User POSTs credentials to /api/auth/login
  2. Backend validates password (bcryptjs), returns JWT + sessionId
  3. Frontend stores token in sessionStorage
  4. All protected API calls send: Authorization: Bearer <token>
  5. authMiddleware.js decodes JWT, attaches req.user (with populated employeeId)
  6. On logout: session invalidated server-side + sessionStorage cleared

  Roles:
  +-- Admin     -> Full access to all modules and API endpoints
  +-- Employee  -> Access controlled by individual permission flags

  Permission Keys (checked via hasPermission() in frontend):
  +-- dashboard
  +-- assetRequest
  +-- requestTracker
  +-- mhDevelopmentTracker
  +-- assetSummary
  +-- employeeMaster
  +-- vendorMaster
  +-- settings

  KEY MODULES - FUNCTIONAL OVERVIEW
================================================================================

  1. DASHBOARD (/api/dashboard -> Dashboard.jsx)
     - Aggregated KPI cards (man-hours, vendors, assets, etc.)
     - Trend charts using Chart.js
     - Activity feeds and summary tables

  2. MH REQUEST MODULE (/mh-requests)
     - CreateMHRequestList: AG Grid table of all MH requests
     - MHRequestForm: Add/Edit form for man-hour requests
     - Status workflow: Created -> Approved / Rejected

  3. REQUEST TRACKER (/request-tracker)
     - Read-only tracker view of MH request statuses
     - Filterable by status, date, department

  4. MH DEVELOPMENT TRACKER (/mh-development-tracker)
     - Tracks development lifecycle through waterfall stages
     - VendorSelectionPopup: assign vendors to items
     - ProjectPlanModal: view/edit associated project plan
     - Linked to ProjectPlan model

  5. ASSET MANAGEMENT (/asset-management-update, /asset-summary)
     - AssetManagementUpdate: editable asset register grid
     - AssetSummary: aggregated asset summary report

  6. EMPLOYEE MASTER (/employee-master)
     - EmployeeMaster: searchable/filterable AG Grid of all employees
     - EmployeeForm: comprehensive add/edit form with photo upload
     - EmployeeView: read-only detail card

  7. VENDOR MASTER (/vendor-master)
     - VendorMaster: AG Grid list of all vendors
     - VendorForm: add/edit vendor details
     - VendorScoring: performance scoring dashboard with analytics
     - VendorLoadingChart: capacity/loading chart visualization

  8. LANDING PAGE (/landing)
     - Public marketing page (no auth required)
     - Sections: Hero, Features, Modules, HowItWorks, Stats, Testimonials, FAQ, CTA
     - Reads live stats from /api/public/stats

  9. SETTINGS (/settings)
     - User profile management
     - Scheduled report configuration (email, frequency, recipients)

  10. EMAIL AND REPORTS
      - emailController + emailService: Nodemailer-based email dispatch
      - reportGenerator: Excel/PDF report builder
      - reportScheduler: node-cron automated scheduled report delivery
